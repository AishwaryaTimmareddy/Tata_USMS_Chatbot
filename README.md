# USMS Saffron Azure Chatbot

Azure-only reference application with a React frontend and FastAPI backend.
It intentionally has no local search, local storage, or mock LLM fallback.

## Azure services used by the code

- Azure OpenAI for embeddings and grounded answer generation
- Azure AI Search for hybrid vector and semantic retrieval
- Azure Blob Storage for approved knowledge documents
- Azure Functions for blob-change-triggered indexing orchestration
- Azure Cosmos DB for conversations, feedback, and query analytics
- Application-managed email/password accounts stored in Azure Cosmos DB
- Azure Key Vault through `DefaultAzureCredential`/Key Vault references
- Azure Monitor and Application Insights for telemetry

The Azure resources and search indexer must be provisioned separately. Until
configuration values are supplied, the API remains available but Azure-backed
operations return `503 Service Unavailable` with the missing settings.

## Project layout

```text
backend/   FastAPI API and Azure adapters
frontend/  React customer chatbot and admin dashboard
azure_function/ Blob-triggered Azure AI Search indexer orchestration
```

## Document ingestion flow

The implementation follows sections 5.4, 5.6.1, 5.6.4, and 5.6.6 of the TSD:

```text
Admin portal
  -> Azure Blob Storage container
  -> Azure Function blob trigger
  -> Azure AI Search Indexer
  -> document cracking, metadata, chunking and enrichment skillset
  -> Azure OpenAI embedding deployment
  -> Azure AI Search text and vector index
```

The FastAPI upload endpoint only places approved documents in Blob Storage.
The Function automatically starts the incremental indexer. The admin
`POST /api/admin/reindex` endpoint remains available as an operational fallback.

## Run the backend

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
uvicorn app.main:app --reload --port 8000
```

## Run the frontend

```powershell
cd frontend
npm install
Copy-Item .env.example .env
npm run dev
```

Open `http://localhost:5173`. The frontend can load before Azure is configured;
the Settings page reports exactly which service variables are missing.

## Search index contract

The Azure AI Search index is expected to expose these fields:

| Field | Type | Purpose |
| --- | --- | --- |
| `id` | `Edm.String` | Key |
| `content` | `Edm.String` | Searchable chunk text |
| `contentVector` | `Collection(Edm.Single)` | Embedding vector |
| `title` | `Edm.String` | Source document title |
| `source` | `Edm.String` | Blob path or source URL |
| `category` | `Edm.String` | Filterable content category |
| `isApproved` | `Edm.Boolean` | Filterable publication flag |
| `pageNumber` | `Edm.Int32` | Optional citation page |

Create an Azure AI Search blob indexer and skillset that cracks documents,
chunks content, generates `text-embedding-3-large` vectors, and maps output to
this contract. Set its name as `AZURE_SEARCH_INDEXER_NAME`.

## Authentication and permissions

Every user must register or sign in before using chat. Passwords are hashed with
Argon2 and user records are stored in the Cosmos DB `users` container. FastAPI
issues signed JWT access tokens, and conversations are stored with the token's
user ID so each account can retrieve only its own history.

The bootstrap administrator is configured through:

```text
BOOTSTRAP_ADMIN_USERNAME=saffronadmin
BOOTSTRAP_ADMIN_EMAIL=aichatbot-admin@usmssaffron.onmicrosoft.com
BOOTSTRAP_ADMIN_PASSWORD=<Key Vault reference>
```

Only this account receives the `admin` role and can upload/delete documents or
run the Azure AI Search indexer. The admin Knowledge Base page also supports
secure document listing, inline viewing, downloading, and deletion without
making the Blob container public. Production passwords and the JWT signing key
are generated by Terraform and supplied through Key Vault references.

Create the Cosmos DB containers with these partition keys:

| Container | Partition key |
| --- | --- |
| `users` | `/id` |
| `conversations` | `/userId` |
| `feedback` | `/userId` |
